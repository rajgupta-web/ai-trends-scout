What Happened This Week

01
Game Changer

Computer Use Lands in the Claude Code CLI

Claude Code v2.1.86 shipped computer use directly in the terminal. Claude can now open native apps, click through UI, test its own changes end to end, and fix what breaks — all from your terminal session. The key unlock: GUI-only apps like iOS simulators, macOS utilities, and desktop tools now have verification loops. Enable it via /mcp, find the computer-use tool, and toggle it on.

Why This Matters

This closes the last "last mile" gap in autonomous coding. Agents could already write and run code — now they can visually verify the result. The pattern of "implement → open simulator → tap through → screenshot → fix regressions" is fully automatable.

Try This Week

Actionable Techniques

Technique 01

Close the Loop with Computer Use

If you ship mobile or desktop apps, enable CLI computer use. Give Claude a specific navigation path:

# Enable in /mcp → computer-use toggle

Implement [your change]. Then open the iOS simulator,
run the app, navigate to Settings → Account,
and confirm the new avatar picker appears.
Screenshot each step and report regressions.
Technique 02

README-Driven Development (The Willison Pattern)

Before opening Claude Code for a new feature, spend 10 minutes writing a tight README section: what it does (1 sentence), exact inputs/outputs, edge cases, and how you'd verify it. Then: "Implement [feature] per the spec in README.md. Write tests first." The README becomes both your spec and Claude's constraint set.

Technique 03

Run /best-of-n Before Hard Refactoring (Cursor 3)

If you use Cursor 3, run your hardest refactoring task on three models simultaneously in isolated worktrees. Review the diffs side by side, accept the cleanest. Takes 2–3x longer but produces meaningfully better results on ambiguous problems.

Refactor the authentication middleware to support
both JWT and API key validation, handling token
expiry gracefully.
The Big Idea

Connecting the Dots

Editorial

The Specialization Gap: Why "One Claude" Will Stop Being Enough

Three things happening this week point in the same direction. Cursor 3 ships /best-of-n — explicitly acknowledging that different models make different tradeoffs and you should run them in parallel. The 3-agent "team" pattern on r/ClaudeAI divides cognitive labor between roles rather than relying on a single agent. And the AutoDream + Dispatch + Channels stack formalizes inter-agent communication as a first-class primitive.

The underlying shift: we are leaving the era of "one smart agent does everything" and entering the era of specialized agent roles with explicit handoffs. This is exactly how high-functioning human teams are organized — and for the same reason.

The models that win won't be the biggest or the cheapest — they'll be the ones best-optimized for their role in a pipeline.

For builders today, the actionable implication is to stop trying to make one Claude Code session handle everything. Break your workflows into stages: a "planning agent" outputs a spec, a "build agent" consumes only that document, a "review agent" checks output against spec. You don't need Dispatch to start — a shared Markdown file between sessions is enough.

Gemma 4's 2B edge model running your reviewer loop locally while Claude handles implementation is already technically feasible. The era of specialized model fleets is earlier than most people think.
