New Feature

/ultrareview — One Command Launches a Parallel Multi-Agent Code Review Pipeline

Claude Code v2.1.111 shipped /ultrareview — a command that spawns a coordinated team of parallel subagents, each auditing a different dimension of your diff simultaneously: security, logic, test coverage, performance, and style. Previously, “AI code review” meant one agent sequentially scanning a file. Now it’s a parallelized pipeline that completes in roughly the same wall-clock time as a single-agent review, with dramatically broader coverage.

Why This Matters

For solo founders and small teams without dedicated reviewers, this is a force-multiplier. A PR that got one pair of AI eyes now gets five, each optimized for a different failure mode. For teams using Claude Code in CI, /ultrareview is a drop-in replacement for the “review before merge” step that doesn’t require a senior engineer to block on.

1-Hour Prompt Cache TTL — 30–50% Cost Drop on Batch Pipelines

Why This Matters

Try This Week

Actionable Techniques

Technique 01

Audit Your Opus 4.7 Migration Cost Before You Switch

Before updating to claude-opus-4-7, run your five most common prompt templates through Simon Willison’s token counter at tools.simonwillison.net/claude-token-counter. Text-heavy prompts: 35–46% more tokens. Code-heavy: 5–10% more. Set your Task Budget ceiling accordingly and recalibrate any billing alerts. Don’t get surprised mid-sprint.

Technique 02

Enable 1-Hour Prompt Caching for Batch Pipelines

If you run Claude in any pipeline where the same large system prompt or context reappears across multiple turns, add this to your environment:

ENABLE_PROMPT_CACHING_1H=true

Then check your API dashboard. Any pipeline where context is reused for >5 minutes of wall-clock time is now cheaper. For overnight batch jobs with dozens of turns against the same context, expect 30–50% reduction on context encoding costs.

Technique 03

Add the “Clarifying Questions First” Pattern to Your CLAUDE.md

Borrow Roblox’s planning-mode pattern. Add this directive to your CLAUDE.md:

# Assumption surfacing
Before beginning any task that would modify
more than 5 files, surface your key assumptions
and ask one clarifying question per assumption.
Proceed only after confirmation.

Adds ~30 seconds of upfront conversation, eliminates the expensive “I built the wrong thing” rebuild loop — IBM’s VAKRA benchmark confirmed this is the most common agent failure type.

The Big Idea

Connecting the Dots

Editorial

The Agentic Middleware Layer Is the Real Battleground

Look at what shipped this week in aggregate: OpenAI Codex adds desktop agent control and 111 integrations. Cursor launches Canvas to make AI output visually navigable. Factory raises $150M on a model-agnostic routing architecture. Roblox builds a full planning-to-playtest pipeline. Claude Code ships /ultrareview, push notifications, and session recap for headless runs.

Every major player is building the same thing: an orchestration layer that sits between a human’s intent and multiple specialized AI tools. The underlying models are becoming commodities faster than anyone predicted — Qwen3.6-35B, a 3B active parameter model, beat Opus 4.7 on an image benchmark this week while running on a laptop.

The differentiation is not in the model — it’s in the workflow intelligence that decides which model to call, when to surface assumptions, how to present output, and when to ask the human.

This is why Factory’s model-agnostic architecture is the right enterprise bet. This is why Cursor Canvas exists — it’s not about better code generation, it’s about better human-AI collaboration UI. This is why /ultrareview matters — it’s not smarter code review, it’s parallel specialization a human can verify.

The builders who win the next two years are not the ones who find the best model — they’re the ones who build the best routing, presentation, and human-in-the-loop architecture on top of increasingly commoditized models. The moat is the workflow, not the weights.

For you specifically: the next time you evaluate an AI tool, stop asking “which model does it use?” and start asking “how does it decide what to do, when to ask me, and how to show me the results?”
