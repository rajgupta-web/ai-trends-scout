Ship It Now

Cursor SDK: Package Your Entire AI Workflow as a Team Plugin

Cursor shipped a public TypeScript SDK on April 29 that lets you programmatically spawn and orchestrate agents with a few lines of code, running on local machines or cloud VMs. The same week, Team Marketplace launched: teams can bundle MCP servers, skills, and subagents into installable plugins with three enforcement modes — Default Off, Default On, or Required. Combined with April 30's Security Review beta (two agents that scan PRs for vulnerabilities and prompt injections, plus a scheduled codebase scanner with Slack integration), Cursor's Enterprise offering is now a genuine platform, not just an IDE.

Why This Matters for Builders

You can now encode your best prompt patterns and tool configurations as a Cursor plugin and push it to your whole team instantly — with enforcement. Think of it as npm for AI workflows. The Security Review agents are particularly useful for small teams without a dedicated security engineer: automated security gates in the existing PR workflow, zero extra tooling.

02
Security

Claude Code Hooks: Deterministic Kill Switch for Autonomous Agents

Claude Code's hooks documentation (now live at code.claude.com) covers five lifecycle event types: shell scripts, HTTP endpoints, MCP tools, LLM prompts, and subagents. Key patterns: a PostToolUse hook that runs a linter on every file touched, and a PreToolUse hook that blocks dangerous commands with exit code 2. For headless / CI usage, hooks are how you move from babysitting the agent to trusting it.

Why This Matters

Encode your project's non-negotiables as hooks, not just instructions. A hook that blocks rm -rf is infinitely more reliable than "please don't delete things."

03
New Feature

OpenAI Codex /goal: Set It and Let It Work Unsupervised

Released in Codex v0.128.0 this week, /goal turns Codex from a turn-by-turn assistant into a persistent agent that pursues a defined objective across sessions, interruptions, and token budget resets. Goals include soft-stop behavior at budget boundaries and are shareable across multi-agent systems. Separately, Codex desktop gained background computer use on macOS — screen watching, clicking, and typing autonomously for testing and frontend iteration.

Why This Matters

Define a goal like "get auth tests to 90% coverage" and Codex keeps working toward it in the background. The first true set-and-forget coding agent pattern.

Technique

Agent Skills as SDLC Workflows: Addy Osmani's Framework

Google Chrome's VP of Engineering published a high-signal post on structuring AI agent skills as markdown files with YAML frontmatter, checkpointed steps, and hard exit criteria. The key innovation: "anti-rationalization tables" — pre-written rebuttals to common excuses agents use to skip essential steps. Skills are organized around six SDLC phases (Define, Plan, Build, Verify, Review, Ship) and hit 99 points on HackerNews within hours of posting.

Why This Matters

Your CLAUDE.md skills files should look like this. A skill with hard exit criteria ("tests must pass before marking done") is far more reliable than prose instructions.

05
Parallel Agents

Cursor 3 /best-of-n: Tournament Selection Native in Your IDE

Cursor 3 (launched April 2, now battle-tested) lets you run multiple AI models on the same task simultaneously via /best-of-n, each on an isolated git worktree. Review outputs side-by-side and pick the best. Combine with /multitask for async parallel subagents and multi-root workspaces for cross-repo changes. Cloud agents continue working after you close your laptop.

Why This Matters

Run Claude, GPT, and Grok on the same refactor, pick the winner. Massive quality uplift for high-stakes code at ~3x the token cost of a single run — worth it on complex changes.

200,000 MCP Servers Exposed to RCE — Anthropic Calls It a Feature

OX Security disclosed that MCP's STDIO transport executes OS commands with no sanitization, affecting all four official SDKs (Python, TypeScript, Java, Rust). Try This Week

Three Things to Try Now

Technique 01

Convert Your CLAUDE.md Into an Addy Osmani-Style Skill File

Instead of prose instructions, restructure your most-used workflows as skill files with YAML frontmatter, numbered steps, and hard exit criteria. Add an "anti-rationalization" section with pre-written rebuttals to the three excuses your agent uses most often to skip steps.

---
name: ship-feature
description: End-to-end workflow for shipping a new feature
model: claude-sonnet-4-6
---

## Step 3: Verify (NON-NEGOTIABLE)
- Run: npm test -- must pass 100%
- Run: npm run lint -- zero errors
- If either fails, fix before marking done

## Do not skip this step because...
- "Tests were passing before my changes" -- run them anyway
- "It's just a small fix" -- small fixes break things too
Technique 02

Run /best-of-n on Your Next High-Stakes Refactor

In Cursor 3, pick a function or module you're not happy with and use /best-of-n to run Claude Sonnet, GPT-4o, and one other model simultaneously. Each runs on its own git worktree — no interference. Review the three diffs side-by-side, pick the best, discard the rest. Budget: 3x the token cost of a single run, with 3x the confidence in the output. Best ROI on any refactor over ~200 lines.

Technique 03

Lock Down Your MCP Server: Three Steps Given the RCE Disclosure

Anthropic will not patch the STDIO vulnerability — defense is on you. Audit every MCP server you're running: (1) check whether you're using STDIO transport with web-facing inputs, (2) add explicit allowlists for any shell commands your server can trigger, (3) run your server in a sandboxed Docker container rather than directly on your workstation or in CI. If you're using Claude Code with an MCP tool in your hooks chain, treat that surface as a privileged execution boundary, not a connector.

The Big Idea

One Thing to Sit With

Editorial

The Stack Is Bifurcating: Orchestration Layer vs. Execution Layer

This week's news reveals a clean split forming in the AI developer stack. On one side: orchestration tools — Cursor SDK, OpenAI /goal, Claude Code subagents, AAIF/MCP — that define what agents should pursue and how they coordinate. On the other: execution tools — background computer use, worktrees, headless Claude Code in CI, cloud agents — that do the actual work without human presence.

The builders winning right now have figured out both layers. Context engineering is how you make orchestration reliable. Hooks and skills are how you make execution safe.

The MCP security story is a reminder that these layers interact in ways that create new attack surfaces. Command execution that would never happen in a human workflow becomes trivially scriptable when agents are involved. Anthropic's "it's a feature" response is technically accurate and strategically problematic simultaneously — it pushes security responsibility to every developer running an MCP server, at a time when 200,000 of them apparently haven't thought about it.

The prediction: within 6 months, "AI workflow engineer" becomes a real job title at companies with more than 20 engineers. Not a developer who uses AI, but someone whose primary job is designing the orchestration layer — the skills, hooks, MCP configurations, and goal architectures that make the execution layer trustworthy enough to run unsupervised. The Cursor Team Marketplace (bundle and enforce agent configs across your org) is an early sign of the tooling that role will use.
